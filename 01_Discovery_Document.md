# Ant Recruit - Discovery Document

## 1. Assumptions

### Business Assumptions
- Ant Recruit is a new recruitment platform entering a competitive market
- Primary revenue model: Employer subscriptions/placement fees
- Target market: Mid-to-senior level professionals and growing tech companies
- Mobile-first approach is critical (most job seekers use mobile)
- Brand positioning: Fast, modern, tech-forward recruitment solution

### User Assumptions
- Job seekers want quick, frictionless application process
- Employers need quality candidates quickly
- Users expect modern, clean UI similar to LinkedIn, Indeed, or Glassdoor
- Mobile app usage will be higher than web for job seekers
- Trust is built through transparency and professional design

### Technical Assumptions
- Website built with responsive design (mobile, tablet, desktop)
- Mobile app available on iOS and Android
- CV upload supports PDF, DOC, DOCX formats
- Integration with Google/LinkedIn for social login
- Real-time application status updates

## 2. Business Goals vs User Goals

### Business Goals
1. **Acquire Job Seekers**
   - Drive mobile app downloads
   - Build user base for employer value proposition
   - Increase application volume

2. **Acquire Employers**
   - Generate qualified leads through contact forms
   - Book discovery calls with potential clients
   - Showcase service offerings and success stories

3. **Build Brand Trust**
   - Establish Ant Recruit as credible recruitment partner
   - Differentiate from competitors
   - Create professional, modern brand perception

4. **Conversion Optimization**
   - Website → App download (job seekers)
   - Website → Contact/Book call (employers)
   - App → Application submission

### User Goals (Job Seekers)
1. **Find Relevant Jobs Quickly**
   - Browse jobs by location, role, salary
   - Save interesting positions
   - Get notified about new matches

2. **Apply Easily**
   - Upload CV once, reuse for multiple applications
   - Quick apply with optional cover note
   - Track application status

3. **Manage Profile**
   - Update CV and personal information
   - View application history
   - Access saved jobs

### User Goals (Employers)
1. **Understand Services**
   - Learn about Ant Recruit's recruitment solutions
   - See pricing/plans (if applicable)
   - View case studies or testimonials

2. **Get Started Quickly**
   - Book a call with minimal friction
   - Submit hiring needs easily
   - Receive quick response

3. **Build Trust**
   - Understand company background
   - See success metrics
   - Read reviews/testimonials

## 3. Key UX Challenges

### Challenge 1: Dual Audience (Job Seekers + Employers)
**Problem:** Website serves two distinct user types with different goals
**Solution:** 
- Clear navigation separation (primary: Job Seekers, secondary: For Employers)
- Distinct CTAs and landing sections
- Separate flows that don't interfere with each other

### Challenge 2: Website → App Conversion
**Problem:** Job seekers browse on web but must download app to apply
**Solution:**
- Show compelling job previews on website
- Clear "Apply on App" CTAs with download badges
- Seamless transition messaging
- QR codes for easy mobile download

### Challenge 3: Mobile App Onboarding
**Problem:** New users need to understand value before signing up
**Solution:**
- 2-3 slide onboarding highlighting key benefits
- Social login options to reduce friction
- Progressive disclosure of features

### Challenge 4: Job Discovery & Filtering
**Problem:** Users need to find relevant jobs without overwhelming options
**Solution:**
- Smart default filters (location, recent jobs)
- Search with autocomplete
- Filter chips for quick refinement
- Save filter preferences

### Challenge 5: Application Flow Friction
**Problem:** Long application forms reduce completion rates
**Solution:**
- Pre-fill from profile/CV
- Optional cover note (not required)
- Progress indicators
- Quick apply option for saved CV

### Challenge 6: Trust Building
**Problem:** New brand needs to establish credibility
**Solution:**
- Professional design language
- Testimonials and case studies
- Transparent process explanations
- Security badges and privacy information

## 4. Target Audience Definitions

### Primary: Job Seekers

**Demographics:**
- Age: 25-45
- Tech-savvy professionals
- Mid-to-senior level roles
- Active mobile users

**Psychographics:**
- Value efficiency and speed
- Want modern, intuitive tools
- Prefer mobile-first experiences
- Expect transparency in process

**Pain Points:**
- Long application processes
- Poor job matching
- Lack of application status updates
- Cluttered job boards

**Goals:**
- Find relevant jobs quickly
- Apply with minimal effort
- Track application progress
- Save jobs for later review

### Secondary: Employers

**Demographics:**
- HR managers, recruiters, founders
- Companies with 10-500 employees
- Tech, finance, marketing sectors
- Mix of desktop and mobile usage

**Psychographics:**
- Need quality candidates fast
- Value efficiency and ROI
- Prefer consultative approach
- Want to understand process before committing

**Pain Points:**
- Slow candidate sourcing
- Poor candidate quality
- High recruitment costs
- Lack of transparency from agencies

**Goals:**
- Understand service offerings
- See pricing/ROI potential
- Book consultation easily
- Build trust before engagement

## 5. Market References

### Direct Competitors (Features to Consider)
1. **LinkedIn**
   - Professional networking focus
   - Clean, corporate design
   - Strong mobile app
   - *Differentiation:* Faster, more focused on job matching

2. **Indeed**
   - Simple job search
   - Aggregated listings
   - *Differentiation:* Curated, quality-focused approach

3. **Glassdoor**
   - Company reviews + jobs
   - Transparent salary info
   - *Differentiation:* Faster application process

### Design References (Visual Inspiration)
1. **Stripe** - Clean, modern, professional
2. **Linear** - Minimal, focused, fast
3. **Notion** - Friendly yet professional
4. **Airbnb** - Trust-building, visual storytelling

### UX Patterns to Adopt
- **Job Cards:** Clear hierarchy (title, company, location, salary, date)
- **Filter Chips:** Quick, visual, removable
- **Application Status:** Visual progress indicators
- **Onboarding:** Benefit-focused, not feature-focused
- **CTAs:** Clear, action-oriented language

- **Trustworthy:** Transparent process, professional design, clear communication

- Professional but approachable

## 7. Success Metrics (Design Impact)

### Website Metrics
- App download conversion rate (target: 15-20%)
- Employer contact form submissions (target: 5-10%)
- Time on site (target: 2-3 min average)
- Bounce rate (target: <40%)

### Mobile App Metrics
- Sign-up completion rate (target: 60-70%)
- Application completion rate (target: 80%+)
- Daily active users
- Jobs saved per user
- Application-to-interview conversion
